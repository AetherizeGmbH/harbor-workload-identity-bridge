WARNING — cluster-name uniqueness (ADR-0009):
  Bridges that share one Harbor must each have a distinct
  `clusterName` that is NOT a hyphen-prefix of any other cluster's
  name. Robots in Harbor are namespaced by name prefix; two clusters
  whose names share a prefix can adopt each other's robots.

  This cluster's prefix is: `bridge-{{ .Values.clusterName }}-`

NEXT STEPS

       kubectl -n {{ .Release.Namespace }} logs deploy/{{ include "harbor-bridge.bridge.fullname" . }} -c bridge

  2. Verify the plugin landed on every node:

       kubectl -n {{ .Release.Namespace }} rollout status ds/{{ include "harbor-bridge.plugin.fullname" . }}

     CR's spec.trustPolicy.audience must equal:

{{- if .Values.plugin.patchKubelet }}

     plugin.patchKubelet=true (current setting): the plugin DaemonSet
     patches /etc/default/kubelet on each node and restarts kubelet
     ONCE per node to apply the flags. Expect a brief node-local
     kubelet bounce as the DaemonSet rolls out — static pods on the
     control plane (apiserver / controller-manager / scheduler) will
     recover within seconds.
{{- else }}

     plugin.patchKubelet=false (current setting): the chart will NOT
     touch kubelet. You must configure the two flags above at node
     provisioning time (kind kubeadm patches, EKS node bootstrap,
     custom AMI, etc.), or the plugin is installed but never invoked.
{{- end }}
